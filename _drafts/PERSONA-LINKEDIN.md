### What's Working Well

- **Headline is strong.** "Engineering Leader" leads, EMBA is visible, all three domains present.
- **Summary/About is strong.** Leads with leadership, hits all five SVP dimensions, ETAAC is
  included, EMBA framing is honest and compelling.
- **Top 5 skills are aligned.** IT Management, IT Strategy, Enterprise Architecture, Machine
  Learning, AWS — reads as technology leader, not senior IC.

### What Needs Attention

#### Experience Descriptions Are IC-Voiced

The biggest gap is in the experience section. Every role description reads as what *you did*,
not what *you led, decided, or enabled*. This is the same Pattern #1 from PERSONA.md
("Lead with Why, Not What") applied to job descriptions.

**Envestnet (current role):**
> "Building large scale cloud platforms for financial services and data science initiatives."

This is IC language. "Building" = doing the work. For SVP positioning, reframe as:
> "Leading cloud platform architecture for financial services, designing the infrastructure
> that supports data science initiatives across the organization. Driving automation-first
> strategy integrating AWS cloud and on-premises data center infrastructure."

**Blue Cross NC:**
> "Building a data science platform incorporating a data-warehouse, data-lake and
> data-execution environment"

Reframe to surface the leadership and impact:
> "Architected and led development of the enterprise data science platform — integrating
> data warehouse, data lake, and containerized execution environment — that powered
> production deep learning models for healthcare predictions. Platform supported the
> CarePath initiative (ISSIP Excellence Award)."

**USPS:**
> "Engaged in advanced analytics with multiple platforms."

"Engaged in" is passive. Reframe:
> "Administered a 25-node SAS Viya in-memory analytics cluster (26TB RAM) linked to a
> Hadoop data lake approaching 1PB. Operated in a high-security federal environment
> processing terabyte-scale datasets for advanced analytics and machine learning."

**NC DIT:**
This one is already the strongest — "Accountable for developing, maintaining and overseeing"
is leadership language. Minor polish only.

**NC DOR / ETAAC:**
These are already strong. The ETAAC description is excellent — "reports to Congress annually"
is a credibility signal most candidates can't match.

#### Missing: ISSIP Excellence Award

The CarePath deep learning work at BCBSNC won an ISSIP Excellence Award. This should be in
the Blue Cross NC description and possibly in Honors & Awards.

#### Missing: BD Biosciences Title Undersells

The PDF says "IT Engineer" but the authorial-voice.md references "IT Director at BD
Biosciences." If the actual title was IT Director (or if you functioned as one — sole IT
person for the plant), the LinkedIn title should reflect that. "Provide for all IT services
at the BD Treyburn manufacturing plant" is director-level scope described with an engineer
title.

#### Certifications Are Dated

The certifications are all 2002-2006 era (Windows 2000, Nortel Networks). They don't hurt,
but they don't help for SVP positioning either. Consider:
- Adding AWS certifications if you have them (or plan to get them)
- The SAS certifications are still relevant for the data science narrative
- The ITIL Foundation cert is fine

#### Early Career Consolidation

The pre-2005 roles (Interpath, Ziff-Davis, Pioneer Software, DB Basics, Roemer Weather,
NC State TA/Library/Operator) add length without adding SVP signal. Consider:
- Consolidating into a single "Early Career" entry: "1990-2005: Progressive roles in systems
  engineering, benchmark development, network operations, and technical services management
  across Ziff-Davis Publishing, Pioneer Software, NetIQ, NC State University, and others."
- This preserves the breadth signal without the recruiter scrolling through 8 pages.

#### Publications Section

Only one publication listed. If you have others (Georgia Tech research, conference papers),
add them. The Google Scholar profile shows publications — make sure LinkedIn matches.
